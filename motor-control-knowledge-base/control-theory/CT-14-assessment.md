## 一、带宽计算与分配（★★★☆☆）

### 题目1：10:3:1带宽法则的相位裕度推导
**问题：** 三环级联系统中，电流环带宽ωc_i=1500 rad/s。
1. 按10:3:1法则，速度环和位置环的带宽应设为多少？
2. 若电流环闭环近似为一阶惯性环节 $G_{ci}(s) = \frac{\omega_{c\_i}}{s+\omega_{c\_i}}$，计算其在速度环截止频率处引入的相位滞后
3. 验证该相位滞后是否保证速度环仍有足够的相位裕度（假设速度环自身设计PM=60°）

**参考答案：**
1. 速度环带宽 ωc_v = 1500/5 = **300 rad/s**（或 1500/10×3 = 450 rad/s，工程常用 1/3~1/5）
   位置环带宽 ωc_p = 300/5 = **60 rad/s**（或 1500/10 = **150 rad/s**）
   
   严格10:3:1 → ωc_i : ωc_v : ωc_p = 1500 : 450 : 150 rad/s

2. 电流环在 ωc_v=450 rad/s 处：
   相位滞后 = -arctan(ω/ωc_i) = -arctan(450/1500) = -arctan(0.3) ≈ **-16.7°**

3. 速度环可用相位裕度 = 60° - 16.7° = **43.3°**
   
   若按更保守的1/5分配（ωc_v=300）：相位滞后=-arctan(0.2)=-11.3°，PM剩余=48.7°✓
   
   结论：10:3:1是保证每层PM≥45°的经验法则的数学表达。

### 题目2：内环非理想性对等效被控对象的影响
**问题：** 电流环实际存在一拍数字延迟Ts=62.5μs（16kHz），其闭环传递函数可近似为：
$$G_{ci}(s) \approx \frac{\omega_{c\_i}}{s+\omega_{c\_i}} e^{-T_s s}$$
其中 ωc_i=2000 rad/s。
1. 计算延迟在速度环带宽 ωc_v=400 rad/s 处引入的额外相位滞后
2. 该延迟对速度环整定有何影响？应如何补偿？

**参考答案：**
1. 延迟引入的相位滞后：$\phi_d = -\omega \cdot T_s = -400 \times 62.5\times10^{-6} = -0.025$ rad ≈ **-1.43°**
   
   总相位滞后（惯性+延迟）= -arctan(400/2000) + (-1.43°) = -11.3° - 1.43° = **-12.73°**

2. 影响：
   - 延迟虽小但不可忽略——在更高带宽比下累积效应显著
   - 速度环PI整定时需额外预留~2~5°相位裕度
   
   补偿方案：
   - 降低速度环目标带宽（如从400降至350 rad/s）
   - 或在速度环控制器中引入Smith预估器/超前补偿抵消延迟
   - 或提高电流环PWM频率（减小Ts）从根本上解决

### 题目3：三环前馈注入点选择
**问题：** 伺服系统采用三环级联结构，需要实现加速度前馈以提升轨迹跟踪性能。
1. 加速度前馈应该注入到哪个环？为什么？
2. 若加速度前馈增益为 Kff_a = J_total（总惯量），写出完整的加速度前馈信号链
3. 如果错误地将加速度前馈注入到位置环输出端，会有什么问题？

**参考答案：**
1. **加速度前馈应注入到电流环给定端（Iq_ref）**。
   
   理由：前馈信号应尽可能靠近执行器注入，绕过所有外层控制器的动态延迟。加速度→转矩→电流（Iq）的关系是直接的物理映射（T=Kt×Iq），在Iq_ref处注入可实现「零延迟」的前馈补偿。

2. 完整信号链：
   ```
   位置指令 θ_ref → 二次微分 → ä_ref
   → Iq_ff = J_total × ä_ref / Kt    （转矩转电流）
   → Iq_total = Iq_ff + Iq_pi_output （前馈+反馈复合）
   ```

3. 错误注入到位置环输出端的问题：
   - 位置环输出的是速度给定 ω_ref，量纲不匹配（加速度 vs 速度）
   - 即使做量纲转换（积分一次），信号仍需经过速度环PI和电流环PI两级延迟
   - 前馈的「零延迟」优势完全丧失，退化为普通的反馈增强
   - 可能导致速度环和电流环给定剧烈波动，引发振荡

### 题目4：三环联调时的稳定性退化分析
**问题：** 工程师分别单独调试了三个环：
- 电流环：ωc_i=1800 rad/s, PM=65°（单独测试）
- 速度环：ωc_v=500 rad/s, PM=60°（假设电流环为理想单位增益）
- 位置环：ωc_p=100 rad/s, PM=55°（假设速度环为理想单位增益）

三环联调后，实测发现速度环出现约8%的超调增大，位置环有低频颤振。
1. 用频域分析方法解释这一现象
2. 给出正确的带宽重新分配方案
3. 计算修正后的各环相位裕度

**参考答案：**
1. 根因分析：
   - **带宽比不足**：ωc_i/ωc_v = 1800/500 = 3.6 < 5（推荐值），速度环频段内电流环不能视为理想单位增益
   - 电流环在 ωc_v=500 处的相位滞后：-arctan(500/1800) = **-15.5°**
   - 速度环实际PM = 60° - 15.5° = **44.5°** ← 超调增大的直接原因
   - 同理，速度环在 ωc_p=100 处的相位滞后进一步恶化位置环稳定性

2. 重新分配方案（按≥5:1原则）：
   - 电流环保持 ωc_i=1800 rad/s（已足够快）
   - 速度环降至 ωc_v=1800/5 = **360 rad/s**（或保守取300 rad/s）
   - 位置环降至 ωc_p=360/5 = **72 rad/s**（或保守取60 rad/s）

3. 修正后相位裕度估算：
   - 速度环：PM_v = 设计PM(60°) - arctan(360/1800) = 60° - 11.3° = **48.7°** ✓
   - 位置环：PM_p = 设计PM(55°) - arctan(72/360) - 速度环在72rad/s处的附加滞后
     ≈ 55° - 11.3° - 约5° ≈ **38.7°** ✓（偏低但可用，可进一步降低ωc_p改善）

### 题目5：级联结构的Anti-windup设计
**问题：** 三环级联系统中，电流环输出Vq受SVPWM限幅±27.7V（Vdc=48V）。当位置环给定位移阶跃时，速度环输出大Iq_ref导致电流环饱和。
1. 说明为何标准条件anti-windup（仅限制积分器输出）在三环结构中不够用
2. 设计一种跨环back-calculation anti-windup方案
3. 写出C代码框架

**参考答案：**
1. 标准anti-windup的问题：
   - 仅限制本环积分器：电流环积分受限，但**速度环和位置环的积分器仍在持续累积**
   - 外环积分器的累积误差通过级联链传递到内环→即使电流环退出饱和，外环积分数值仍然偏大→再次触发饱和→**振荡循环**
   - 需要**跨环协调**：当内环饱和时，通知外环也进行anti-windup处理

2. 跨环back-calcation方案：
   ```
   当 |Vq_out| > Vq_max 时:
     sat_err = Vq_out - saturate(Vq_out, Vq_max)
     电流环积分器 -= K_aw_i × sat_err      // 本环反算
     向上传递: speed_AW_err = sat_err × K_iq_to_speed  // 换算到速度环量纲
     速度环积分器 -= K_aw_v × speed_AW_err // 速度环反算
     (可选) 继续向上传递到位置环
   ```

3. C代码框架：
   ```c
   // 电流环执行后
   float Vq_sat = clamp(Vq_pi + Vq_ff, -VQ_MAX, VQ_MAX);
   float Vq_err = (Vq_pi + Vq_ff) - Vq_sat;
   
   if (fabs(Vq_err) > 0.001f) {
       pi_q.integral -= AW_K_I * Vq_err;  // 本环AW
       
       float speed_aw_err = Vq_err / Kt;   // 转换为速度环量纲
       pi_w.integral -= AW_K_W * speed_aw_err;  // 跨环AW
   }
   ```

### 题目6：模式切换的Bumpless Transfer
**问题：** 伺服驱动器需要支持力矩模式 ↔ 速度模式 ↔ 位置模式的无扰切换。
1. 分析切换时产生冲击的根本原因
2. 设计完整的三模式bumpless transfer方案
3. 给出C代码实现的关键步骤

**参考答案：**
1. 冲击原因：
   - 不同模式下控制器结构不同：力矩模式只有电流环；速度模式增加速度PI；位置环再增加位置PI
   - 切换瞬间新激活的控制器积分器初值为0（或未初始化），而当前跟踪误差不为零
   - 积分器从零开始累积 → 输出跳变 → 内环给定突变 → 电机冲击

2. Bumpless Transfer方案：
   ```
   切换到新模式时:
   a) 计算新模式下的稳态控制输出（基于当前状态）
   b) 将新激活控制器的积分器初始化为: u_integral = u_steady_state - Kp × error
   c) 使切换瞬间总输出连续（无跳变）
   d) 可选: 在N个周期内线性过渡
   ```

3. C代码关键步骤：
   ```c
   void switch_to_position_mode(void) {
       float current_pos = get_encoder_position();
       float pos_error = pos_ref - current_pos;
       
       // 速度环: 初始化积分器使输出=当前速度给定
       float spd_current = get_speed_feedback();
       pi_w.integral = spd_current - pi_w.Kp * 0;  // 速度误差≈0
       
       // 位置环: 初始化积分器使输出=当前速度给定
       pi_p.integral = spd_current - pi_p.Kp * pos_error;
       
       mode = POSITION_MODE;
   }
   ```

## 三、综合设计与优化（★★★★☆）

### 题目7：前馈补偿的三环协同设计
**问题：** 一台伺服电机（J=0.002 kg·m², Kt=0.5 Nm/A）需要执行S曲线轨迹跟踪。
1. 分别给出速度前馈和加速度前馈的注入点和增益公式
2. 分析前馈对各级环的影响——是否需要调整原有PI参数？
3. 当前馈模型不准（惯量辨识误差±20%）时，系统的鲁棒性如何？

**参考答案：**
1. 前馈注入点与增益：
   | 前馈类型 | 注入点 | 增益公式 |
   |---------|--------|---------|
   | 速度前馈FFv | 速度环输出端（与速度PI输出相加） | Kff_v = 1.0（纯单位前馈） |
   | 加速度前馈FFa | 电流环给定端（与电流环Iq_ref相加） | Kff_a = J/Kt = 0.002/0.5 = 0.004 A·s²/m |

2. 对PI参数的影响：
   - **理想情况下不需要调整PI参数**：前馈是开环补偿通路，不影响闭环极点
   - **实际建议适当降低PI增益**：前馈承担了大部分跟踪任务，PI只需处理残差→可降低Kp/Ki提高鲁棒性
   - 典型做法：PI增益降至原来的50~70%，前馈承担70~90%的控制量

3. 鲁棒性分析（惯量误差ΔJ/J=±20%）：
   - 前馈转矩误差 = ΔJ × ä → 残余扰动由反馈PI消除
   - 若J估计偏小20%：前馈欠补偿→PI需多补偿20%→稳态误差略有增加（由积分项消除）
   - 若J估计偏大20%：前馈过补偿→PI反向纠偏→可能引起轻微超调
   
   结论：**前馈+反馈的复合架构天然具有鲁棒性**——前馈模型不准时，反馈自动接管补偿。这是纯前馈控制无法比拟的优势。

### 题目8：谐振频率约束下的带宽设计
**问题：** 伺服系统的机械传动链存在谐振频率 ω_res = 400 Hz ≈ 2513 rad/s。
1. 位置环和速度环的带宽应如何避开此谐振频率？
2. 若电流环带宽必须设在2000 rad/s（接近谐振频率），应采取什么措施？
3. 给出完整的带宽分配方案（考虑谐振约束）

**参考答案：**
1. 谐振避让原则：
   - 各环的截止频率应 < ω_res/3 ~ ω_res/5（避免激发谐振）
   - 速度环带宽上限：2513/5 ≈ **500 rad/s**（保守）或 2513/3 ≈ **838 rad/s**（激进）
   - 位置环带宽上限：若速度环取500 rad/s，则位置环 ≤ **100 rad/s**

2. 电流环接近谐振时的措施：
   - **陷波滤波器（Notch Filter）**：在电流环或速度环中加入陷波器，中心频率=f_res，衰减谐振峰值
   - **降低电流环带宽**：牺牲电流响应速度换取稳定性（如降至1200 rad/s）
   - **机械改进**：增加刚性、加装阻尼器（治本之策）

3. 完整带宽分配方案（含谐振约束）：
   ```
   约束条件:
   - ω_res = 2513 rad/s (400Hz)
   - 电流环 ≤ ω_res/2 = 1256 rad/s（留安全裕度）
   - 速度环 ≤ min(电流环/5, ω_res/5) = min(251, 503) = 251 rad/s
   - 位置环 ≤ 速度环/5 = 50 rad/s
   
   推荐方案:
   - 电流环: ωc_i = 1200 rad/s (191 Hz)
   - 速度环: ωc_v = 250 rad/s (40 Hz)  
   - 位置环: ωc_p = 50 rad/s (8 Hz)
   
   带宽比: 24 : 5 : 1（满足≥5:1要求且避开谐振）
   ```

## 四、深度思考题（★★★★★）

### 题目9：三环级联 vs 单环状态反馈的对比
**问题：** 对于同一个伺服位置控制问题，有两种方案：
- 方案A：传统三环级联PID（电流PI + 速度PI + 位置PID/P）
- 方案B：全状态反馈（测量位置、速度、电流，单环LQR/LQG）

请从以下维度对比两种方案：
1. 参数整定复杂度
2. 对传感器噪声的敏感性
3. 故障诊断与隔离能力
4. 工业界采纳度及原因

**参考答案：**

| 维度 | 方案A：三环级联 | 方案B：全状态反馈 |
|------|---------------|------------------|
| **参数整定** | 分层独立整定，每环2个参数，共6个。有成熟的带宽法则指导。物理意义明确。 | 需要整定Q/R矩阵（多维权重），需解Riccati方程。参数耦合，物理意义不直观。 |
| **噪声敏感度** | 内环带宽高→对电流噪声敏感但外环自然滤除。分层滤波：电流环快速响应高频，位置环自然低通。 | 所有增益同时作用，状态反馈将所有测量噪声直接映射到控制量。需仔细设计Kalman滤波器权衡响应速度与噪声抑制。 |
| **故障隔离** | 天然分层：电流环故障不影响位置环逻辑。可在任一环加入独立限幅和保护。易于定位故障层级。 | 单环结构：任一传感器故障直接影响整体控制。故障隔离需额外的FDI（故障检测与隔离）逻辑。 |
| **工业采纳度** | ★★★★★ 极高。几乎所有商用伺服驱动器（西门子、安川、松下等）均采用三环级联。 | ★★☆☆ 低。主要存在于学术研究和特殊应用（航空航天、高精度半导体设备）。 |

**工业界偏好三环级联的原因**：
1. **模块化**：每个环可独立开发、测试、验证
2. **可调试性**：工程师可逐环调试，问题定位清晰
3. **继承性**：从单速控制（仅电流环）到伺服控制（三环）平滑扩展
4. **安全性**：每层都有独立的限幅和保护机制
5. **标准化**：国际标准（如IEC 61800-4）定义了三环接口

**方案B的优势场景**：
- 多变量强耦合系统（如多轴机械臂的动力学控制）
- 需要最优性能且计算资源充足（航天、国防）
- 已有精确模型且参数变化小

### 题目10：实时性约束下的三环调度策略
**问题：** 一个完整的FOC伺服中断服务程序包含以下任务（按优先级排序）：
1. ADC采样 & 电流环 PI（必须在PWM周期内完成）
2. 速度观测 & 速度环 PI（可与下一拍电流环并行）
3. 位置读取 & 位置环 PI / P（周期可更长）
4. 通信 & 故障检测（最低优先级）

已知：PWM频率16kHz（Ts=62.5μs）， Cortex-M4 @168MHz，FOC核心算法耗时8μs。
1. 设计合理的多速率调度策略
2. 若位置环采用4倍电流环周期（Ts_pos=250μs），对位置跟踪精度有何影响？
3. 如何在资源有限的MCU上平衡三环的计算负载？

**参考答案：**
1. 多速率调度策略：

| 任务 | 执行周期 | 优先级 | 耗时预算 |
|------|---------|--------|---------|
| ADC采样+电流环 | 62.5μs (16kHz) | 最高 | ≤15μs |
| 速度环 | 125μs~250μs (8k~4kHz) | 高 | ≤10μs |
| 位置环 | 250μs~1ms (4k~1kHz) | 中 | ≤10μs |
| 通信/保护 | 1ms~10ms | 低 | 余量 |

2. 位置环4倍周期的精度影响：
   - **位置跟踪延迟增加**：最坏情况下位置校正延迟从62.5μs增至250μs
   - **对恒速跟踪影响小**：位置误差 = v × Ts_delay，对于1000rpm（105rad/s），误差≈105×250μs=0.026rad≈1.5°
   - **对动态轨迹有影响**：S曲线加减速段的位置跟踪误差增大
   - **缓解措施**：位置环内部采用插值或预测补偿

3. MCU资源平衡策略：
   ```c
   void PWM_IRQHandler(void) {  // 16kHz
       adc_sample();            // 1μs
       clarke_park();           // 1μs
       current_pi_update();     // 3μs
       ipark_svpwm();           // 2μs
       pwm_reload();            // 1μs
       // 总计 ~8μs << 62.5μs ✓
   }
   
   void MediumTask(void) {  // 4kHz (每4个PWM周期触发)
       speed_estimate();    // 2μs (滑模观测器)
       speed_pi_update();   // 2μs
       Iq_ref_limit_check();// 0.5μs
   }
   
   void SlowTask(void) {    // 1kHz
       position_read();     // 1μs
       position_p_update(); // 2μs
       comm_handler();      // 余量
   }
   ```

   关键优化：
   - 电流环在中断中同步执行（硬实时）
   - 速度环可在主循环中以"就绪标志"方式执行（软实时）
   - 位置环可进一步降频或使用RTOS低优先级任务
   - 使用DMA减轻CPU负担（ADC、编码器接口）
