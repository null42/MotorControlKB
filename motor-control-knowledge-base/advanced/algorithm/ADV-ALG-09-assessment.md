## 第1题：标幺值基值选取的一致性原则

**题目：** 在标幺值系统中，以下关于基值选取的说法正确的是？

A. 电压基值、电流基值、阻抗基值可以独立选取，互不影响
B. 只能独立选取两个基值（如 $V_{base}$ 和 $I_{base}$），其余基值必须由这两个派生
C. 所有基值都必须独立选取，以保证精度
D. 基值选取不需要满足量纲一致性，只要数值方便即可

**正确答案：** B

**详细解析：**

标幺值系统的核心约束是**量纲一致性**。所有派生基值必须从基本基值出发推导：

$$Z_{base} = V_{base}/I_{base}, \quad \omega_{base} = V_{base}/\psi_f, \quad L_{base} = Z_{base}/\omega_{base}$$

如果独立选取三个或更多基值，必然出现不一致。例如，如果独立选取 $V_{base}$、$I_{base}$ 和 $Z_{base}$，则 $Z_{base}$ 可能不等于 $V_{base}/I_{base}$，导致标幺值下的欧姆定律 $V_{pu} = I_{pu} \times Z_{pu}$ 不成立。

**验证方法：** 检查以下关系是否全部满足：
- $V_{base} = I_{base} \times Z_{base}$
- $Z_{base} = \omega_{base} \times L_{base}$
- $V_{base} = \omega_{base} \times \psi_{base}$

## 第2题：转速基值的物理意义

**题目：** 在PMSM标幺值系统中，转速基值 $\omega_{base} = V_{base}/\psi_f$ 的物理意义是？

A. 电机的额定转速
B. 电机的最大允许转速
C. 反电动势等于电压基值时的电角速度，即弱磁控制的起始点
D. 电流达到基值时的转速

**正确答案：** C

**详细解析：**

当 $\omega_e = \omega_{base}$ 时：

$$E = \omega_e \psi_f = \omega_{base} \times \psi_f = \frac{V_{base}}{\psi_f} \times \psi_f = V_{base}$$

即反电动势标幺值为1.0。这正是弱磁控制的起始点——当反电动势达到电压基值时，电压裕度为零，必须开始弱磁。

这一设计的巧妙之处在于：标幺值下 $\omega_{e,pu} = 1.0$ 恰好对应弱磁起始点，物理意义极其清晰。如果 $\omega_{e,pu} < 1$，电压有余量，不需要弱磁；如果 $\omega_{e,pu} > 1$，电压饱和，必须弱磁。

同时，磁链基值 $\psi_{base} = V_{base}/\omega_{base} = \psi_f$，因此标幺值下 $\psi_{f,pu} = 1.0$，这也是一个必然结果。

## 第3题：Q14格式选择的原因

**题目：** MC_LIB使用Q14格式而非Q15格式作为主要定点格式，主要原因是？

A. Q14的计算速度比Q15快
B. Q14的1.0对应16384，留出了1位余量空间，允许标幺值超过1.0（如过调制时电压可达1.15 PU）
C. Q14的精度比Q15高
D. Q14是ARM CMSIS-DSP库的标准格式

**正确答案：** B

**详细解析：**

Q15的1.0对应32767，表示范围为 $[-1, 0.99997]$，超过1.0就溢出了。Q14的1.0对应16384，表示范围为 $[-2, 2 - 2^{-14}]$，标幺值可以超过1.0。

在电机控制中，标幺值超过1.0的情况并不罕见：
- **过调制**时电压可达 $1.103\text{ PU}$（六步方波比线性调制多10.3%）
- **瞬态超调**时电流可能短暂超过1.0 PU
- **弱磁区**某些中间计算值可能超过1.0

Q14留出了1位的余量空间，允许标幺值在 $[-2, 2)$ 范围内安全表示，避免了溢出风险。

Q14的精度（$2^{-14} = 6.1 \times 10^{-5}$）比Q15（$2^{-15} = 3.05 \times 10^{-5}$）略低，但对电机控制来说完全足够。CMSIS-DSP的标准格式是Q15和Q31，不是Q14。

## 第4题：Q格式乘法运算

**题目：** 在Q14定点运算中，两个Q14数相乘的结果是什么格式？如何转换回Q14？

A. 结果是Q14，直接使用
B. 结果是Q28，需要右移14位转换回Q14
C. 结果是Q28，需要左移14位转换回Q14
D. 结果是Q7，需要左移7位转换回Q14

**正确答案：** B

**详细解析：**

Q格式乘法规则：$Q_a \times Q_b = Q_{a+b}$

Q14 $\times$ Q14 = Q28，因为小数位数相加：14 + 14 = 28。

**具体示例：**
```c
Q32I_ a_q14 = 8192;    // 0.5 in Q14 (8192/16384 = 0.5)
Q32I_ b_q14 = 4096;    // 0.25 in Q14 (4096/16384 = 0.25)
Q32I_ result_q28 = a_q14 * b_q14;  // 33554432 = 0.125 in Q28
Q32I_ result_q14 = result_q28 >> 14;  // 2048 = 0.125 in Q14
```

验证：$0.5 \times 0.25 = 0.125$，正确。

右移14位（$>>14$）是截断操作，将Q28的小数位从28位减少到14位。左移会增大数值，方向错误。

**注意：** 多个Q28结果累加时可能溢出32位，需要使用64位累加器。

## 第5题：标幺值下PI参数的物理意义

**题目：** 在标幺值下，电流环PI参数 $K_{p,pu} = \omega_{bw,pu} \cdot L_{s,pu}$，其中 $L_{s,pu}$ 通常在什么范围？

A. $L_{s,pu}$ 通常远大于1，因为电感值很大
B. $L_{s,pu}$ 通常在 $[0, 1]$ 范围内，因为 $L_{base}$ 通常大于实际电感
C. $L_{s,pu}$ 恒等于1.0，因为电感基值等于实际电感
D. $L_{s,pu}$ 可以为任意值，没有典型范围

**正确答案：** B

**详细解析：**

以文档中的24V/8A PMSM为例：

$$L_{s,pu} = \frac{L_s}{L_{base}} = \frac{0.385\text{mH}}{1.093\text{mH}} = 0.352$$

$L_{s,pu}$ 在 $[0, 1]$ 范围内，因为 $L_{base} = \psi_f / I_{base}$，而实际电感 $L_s$ 通常小于 $\psi_f / I_{base}$（否则电机的短路电流 $I_{sc} = \psi_f / L_s$ 会小于基值电流，这在大多数电机中不成立）。

标幺值下PI参数的优势正在于此：
- 有名值下，不同电机的 $K_p$ 可能从0.01到100差异巨大
- 标幺值下，$K_{p,pu} = \omega_{bw,pu} \times L_{s,pu}$，由于 $L_{s,pu}$ 在 $[0, 1]$ 范围，不同电机的 $K_{p,pu}$ 在同一量级

注意：$\psi_{f,pu} = 1.0$ 是必然结果（因为 $\psi_{base} = \psi_f$），但 $L_{s,pu}$ 不恒等于1.0。

## 第6题：定点数溢出的危害

**题目：** 在Q14定点运算中，如果两个接近最大值的Q14数相加，结果用int16_t存储，最严重的后果是？

A. 结果被截断，丢失低位精度
B. 结果溢出导致符号翻转，正数变负数，控制器输出反向
C. 结果被饱和到最大值，不会产生严重后果
D. 结果自动扩展为32位，没有问题

**正确答案：** B

**详细解析：**

Q14格式下int16_t的最大值为16383（约0.999 PU）。如果两个16383相加：

$$16383 + 16383 = 32766$$

这在int16_t范围内（最大32767），没有溢出。但如果值更大：

$$16384 + 16384 = 32768$$

int16_t的32768变成 $-32768$！**符号翻转**！原本0.5 + 0.5 = 1.0的电压给定，变成了一个巨大的负值，导致逆变器输出反向电压，可能触发过流保护甚至损坏硬件。

这就是为什么定点数运算中**饱和处理**是安全网：

```c
// 安全做法：使用int32_t中间结果 + 饱和
int32_t sum = (int32_t)a_q14 + (int32_t)b_q14;
int16_t result = (int16_t)MATH_SAT_T(sum, 16383, -16384);
```

选项A描述的是精度损失，不是溢出。选项C只有在主动添加饱和处理时才成立。选项D不会自动发生。

## 第7题：CORDIC算法的核心优势

**题目：** CORDIC算法在电机控制中的核心优势是？

A. 计算速度比查表法更快
B. 只需要移位和加法迭代，不需要乘法器，适合无硬件乘法器的MCU
C. 精度比浮点运算更高
D. 可以同时计算所有三角函数

**正确答案：** B

**详细解析：**

CORDIC（COordinate Rotation DIgital Computer）的核心思想是将旋转角度分解为一系列固定角度的逐次逼近，每次迭代只需要**移位和加法**：

$$x_{i+1} = x_i - d_i \cdot y_i \cdot 2^{-i}$$
$$y_{i+1} = y_i + d_i \cdot x_i \cdot 2^{-i}$$

其中：
- $x_i$、$y_i$：第$i$次迭代的坐标分量
- $d_i$：第$i$次迭代的旋转方向，取值为$\pm 1$
- $i$：迭代序号，从0开始

$2^{-i}$ 的乘法可以通过右移 $i$ 位实现，完全不需要乘法器。这对Cortex-M0/M3等没有硬件乘法器或乘法器周期较长的MCU极为重要。

**CORDIC vs 查表法对比：**
- 查表法速度更快（2~5周期 vs 100~200周期软件CORDIC），但占用Flash大
- CORDIC占用Flash极小（几十字节的arctan表），但速度较慢
- STM32G4有硬件CORDIC，4个时钟周期完成，兼具速度和省Flash的优势

选项A错误：软件CORDIC比查表法慢。选项C错误：CORDIC精度取决于迭代次数，通常不如浮点。选项D夸大了，CORDIC可以计算sin/cos/arctan等，但不是"所有"三角函数。

## 第8题：浮点转定点的调试策略

**题目：** 将浮点FOC算法转换为定点实现时，推荐的调试策略是？

A. 一次性将所有浮点运算替换为定点运算，然后整体测试
B. 先浮点验证通过，再逐模块转定点，每一步都对比浮点和定点的输出误差
C. 直接写定点代码，不需要浮点验证
D. 只需验证最终输出，中间过程不需要对比

**正确答案：** B

**详细解析：**

逐步转换法是浮点转定点的标准工程实践：

1. **先浮点验证通过**——确保算法本身正确
2. **确定变量范围和精度**——在浮点代码中添加范围检测，运行典型工况
3. **选择Q格式**——根据范围和精度要求
4. **逐模块替换**——按 Clarke → Park → PI → SVPWM → 观测器的顺序
5. **每替换一个模块，立即对比浮点和定点的输出误差**

每步验证的可接受误差范围：
- 电流/电压（标幺值）：$< 0.001$
- 角度：$< 0.01^\circ$
- 占空比：$< 0.01\%$

选项A一次性替换是最危险的做法——出问题时无法定位是哪个模块的Q格式选择错误。选项C跳过了验证步骤。选项D忽略了中间过程，可能累积误差。

## 第9题：定点数精度损失累积

**题目：** 在定点数运算中，多次乘法后右移截断会导致精度损失累积。以下哪种做法可以有效减少累积误差？

A. 每次乘法后立即右移截断到目标Q格式
B. 延迟截断，在多步乘法完成后只做一次截断
C. 使用更小的Q格式（如Q8代替Q14）
D. 增加采样频率

**正确答案：** B

**详细解析：**

**问题示例——每步截断：**
```c
Q32I_ a = (x1_q14 * y1_q14) >> 14;  // 第1次截断，误差e1
Q32I_ b = (a * y2_q14) >> 14;       // 第2次截断，误差e1+e2
Q32I_ c = (b * y3_q14) >> 14;       // 第3次截断，误差e1+e2+e3
```

每次截断引入约 $\pm 0.5$ LSB的误差，多次截断后误差逐步累积。

**解决方案——延迟截断：**
```c
// Q14 × Q14 × Q14 = Q42，只在最后一步截断
Q32I_ tmp_q42 = x1_q14 * y1_q14 * y2_q14;  // 注意：需要64位中间结果
Q32I_ c = tmp_q42 >> 28;                     // 一步截断到Q14
```

延迟截断只在最后一步引入一次舍入误差，中间过程保持高精度。

**但需注意：** 延迟截断要求中间结果有足够的位宽。Q14 × Q14 × Q14 = Q42，需要64位整数存储。如果位宽不够，仍然需要分步截断，但可以尽量减少截断次数。

选项C使用更小Q格式会降低精度，适得其反。选项D增加采样频率与定点精度无关。

## 第10题：何时必须使用定点数

**题目：** 以下哪种场景**必须**使用定点数而非浮点数？

A. STM32G4（Cortex-M4F，有单精度FPU）上实现20kHz FOC电流环
B. STM32F103（Cortex-M3，无FPU）上实现10kHz FOC电流环
C. 开发调试阶段，需要快速验证算法正确性
D. STM32H7（Cortex-M7，有双精度FPU）上实现5kHz FOC电流环

**正确答案：** B

**详细解析：**

必须使用定点数的场景：

1. **无FPU的MCU**：STM32F103（Cortex-M3）没有硬件浮点单元，浮点运算需要软件模拟，一次浮点乘法约20个时钟周期，而定点乘法只需2个周期。10kHz FOC电流环在100us内需要完成Clarke、Park、PI、反Park、SVPWM等运算，软浮点根本来不及。

2. **极高实时性要求**：即使有FPU，电流环周期 < 50us时，定点仍可能更快。

3. **代码空间受限**：软浮点需要链接浮点库（~8KB），定点只需基本运算（~2KB）。

对于选项A（STM32G4有FPU），浮点运算1个时钟周期，20kHz电流环完全可以用浮点。选项C开发调试阶段应优先用浮点，更直观易调试。选项D的STM32H7性能充裕，浮点毫无压力。

**决策原则：** 有FPU且实时性要求不极端时，浮点的开发效率优势更明显；无FPU时，定点是唯一选择。
