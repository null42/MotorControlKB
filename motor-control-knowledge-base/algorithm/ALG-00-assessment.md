## 第一部分：概念理解题（选择题，每题2分，共20分）

### 题1：时间常数的物理意义

电机绕组的时间常数 τ = L/R = 5 ms，这意味着：

A. 电流在5ms后达到稳态值的100%
B. 电流在5ms后达到稳态值的63.2%
C. 电流在15ms（3τ）后才达到稳态值
D. 时间常数与电感成反比

**答案：B**
**解析：** 根据定义，t=τ时电流达到(1-e⁻¹)≈63.2%。A错误（是63.2%不是100%）；C虽然描述正确（3τ确实接近稳态），但不是τ本身的含义；D错误（τ与L成正比）。

### 题2：零极点对消的条件

为了实现零极点对消，PI控制器的参数应该满足：

A. Kp = Ki × (L/R)
B. Ki/Kp = R/L
C. Kp/Ki = R/L
D. Kp × Ki = R/L

**答案：B**
**解析：** PI零点 s_z = -Ki/Kp，对象极点 s_p = -R/L，令二者相等得到 Ki/Kp = R/L。

### 题3：带宽α增大时的影响

如果将电流环带宽从1000 rad/s增加到3000 rad/s，以下说法**错误**的是：

A. Kp和Ki都会增大
B. 系统响应速度变快
C. 抗干扰能力增强
D. 对噪声更敏感

**答案：C**
**解析：** 带宽增大→系统响应变快（B正确）→Kp=αL, Ki=αR都增大（A正确）→高频噪声被放大（D正确）。但抗干扰能力会**减弱**而非增强，因为高频扰动也会被跟踪。所以C错误。

### 题4：PWM频率约束

某系统PWM频率为10 kHz，理论上电流环最大带宽不应超过：

A. 6280 rad/s (1000 Hz)
B. 3140 rad/s (500 Hz)
C. 62800 rad/s (10000 Hz)
D. 1570 rad/s (250 Hz)

**答案：A**
**解析：** 根据经验法则 ω_bw ≤ 2πf_PWM / 10 = 2π×10000/10 = 2000π ≈ 6280 rad/s。选择1/10的安全裕度是最常见的做法。

### 题5：比例增益Kp的作用

Kp = α·L 中，Kp与电感L成正比的原因是：

A. 电感越大，需要的驱动力越小
B. 电感越大，电磁惯性越大，需要更大的增益克服惯性
C. 电感越大，电阻也越大
D. 这只是巧合，没有物理意义

**答案：B**
**解析：** 电感代表"电磁惯性"，L越大→惯性越大→需要更大的"力"（电压）来获得相同的加速度（di/dt）。这类似于推动质量大的物体需要更大的力。

### 题6：积分增益Ki的物理功能

积分项 Ki 的主要作用是：

A. 加快系统的动态响应速度
B. 消除稳态误差
C. 抑制高频噪声
D. 增加系统阻尼

**答案：B**
**解析：** 积分器会累积历史误差，只要存在稳态误差，积分输出就会持续增长直到误差为零。这是PI控制器能实现无静差跟踪的关键。A是P项的功能；C需要滤波器；D通常需要微分项或调整P参数。

### 题7：一阶系统特征

零极点对消后的闭环系统是一个一阶系统，其阶跃响应特点是：

A. 有超调和振荡
B. 无超调，单调上升
C. 有稳态误差
D. 响应时间与初始条件无关

**答案：B**
**解析：** 一阶惯性环节的阶跃响应是指数上升曲线，单调递增无超调（B正确）。无稳态误差是因为有积分环节（I项）；响应时间由时间常数决定，与初始条件无关（D也正确，但B是最显著的特征）。

### 题8：实际工程中的偏差原因

按理论公式 Kp=αL, Ki=αR 计算的参数，在实际系统中效果不理想，最不可能的原因是：

A. L和R参数测量不准确
B. 数字控制系统存在延迟
C. 公式本身推导有误
D. 存在未建模的干扰（如反电动势）

**答案：C**
**解析：** 零极点对消法的数学推导是严谨的（C不太可能出错）。实际问题通常是：参数不准（A）、延迟影响（C）、未建模扰动（D）等工程因素。

### 题9：调试时的阶跃响应测试

理想的电流环阶跃响应（从0A跳变到额定电流的30%）应该是：

A. 快速上升，超调<5%，快速稳定
B. 极快上升，允许20%超调
C. 缓慢上升，完全无超调
D. 有明显振荡后再收敛

**答案：A**
**解析：** 电流环作为最内环，要求快速且稳定。理想指标：上升时间短、超调小（<5%~10%）、无振荡。B超调过大；C太慢；D不稳定。

- PMSM电机：Ld = Lq = 8 mH, Rs = 0.6 Ω
- PWM频率：f_PWM = 20 kHz
- 选择带宽安全裕度因子：12（即带宽取 f_PWM/12）

**求解：**
(1) 电气时间常数 τ = ? （2分）
(2) 设计带宽 α = ? （rad/s 和 Hz）（2分）

(4) 闭环后的时间常数及提速比（2分）

### 题14：故障诊断与分析（8分）

**场景：** 某工程师设计了电流环参数（基于L=3mH, R=0.4Ω, f_PWM=16kHz, 取α=2πf/10），但在实际测试中遇到以下现象：

**现象记录：**
1. 静止状态下阶跃响应正常（无超调、无振荡）
2. 电机转速达到额定转速的50%时，q轴电流出现±15%的纹波
3. 转速达到80%时，电流波形严重畸变，偶尔触发过流保护

**问题：**
(1) 计算原始设计参数（2分）
(2) 分析可能的原因（至少列出2个，并说明机理）（4分）
(3) 提出解决方案（2分）

**参考答案：**

**(1) 原始参数计算（2分）**

α = 2π×16000/10 = **10053 rad/s** （0.5分）
Kp = 10053×0.003 = **30.16 V/A** （0.5分）
Ki = 10053×0.4 = **4021.2 V/(A·s)** （0.5分）
τ_closed = 1/10053 = **0.099 ms** （0.5分）

**(2) 可能的原因分析（4分，每个原因2分）**

**原因1：反电动势扰动未补偿（2分）**
- 机理：高速旋转时，永磁体切割绕组产生反电动势 e = ω_e × ψ_f
- 这个反电动势相当于叠加在电机端电压上的强扰动
- 低速时e很小可忽略，高速时e很大（可能达到几十伏）
- PI控制器需要额外输出来抵消这个扰动，导致工作点偏移、电流畸变

**原因2：dq轴交叉耦合效应（2分）**
- 机理：dq坐标系下存在交叉耦合项：
  - v_d受ω_e×L_q×i_q影响
  - v_q受ω_e×L_d×i_d影响
- 这些耦合项与转速ω_e成正比，低速时可忽略，高速时不可忽视
- 未解耦会导致两轴相互干扰，电流波形畸变

**其他合理答案（额外加分项）：**
- 电感参数随工作点变化（磁饱和导致L下降，破坏零极点对消条件）
- 采样延迟在高速时影响更大（同样的时间延迟对应更大的电角度误差）
- 死区效应在高速大电流时更明显

**(3) 解决方案（2分，每个方案1分，答对2个即得满分）**

**方案1：加入前馈解耦（必须）**
```
v_d* = PI_d(i_d_error) - ω_e × L_q × i_q
v_q* = PI_q(i_q_error) + ω_e × (L_d × i_d + ψ_f)
```
这样PI控制器只需处理动态误差，稳态前馈项直接抵消耦合和反电动势。

**方案2：适当降低带宽（辅助措施）**
将α从10053降到6000-8000，牺牲一些响应速度换取更好的稳定裕度。

**方案3：在线参数辨识（高级方案）**
实时辨识L和R的变化，自适应调整PI参数，保持零极点对消条件。

### 题15：综合设计题（8分）

**场景：** 你需要为一台新电机设计电流环PI参数。已知以下信息：

**电机铭牌数据：**
- 类型：表贴式PMSM（SPMSM）
- 额定功率：750W
- 额定电压：48V DC
- 额定电流：20A（峰值）
- 额定转速：3000 rpm
- 极对数：4

**实验测量值（室温25°C）：**
- 相电阻 Rs = 0.22 Ω
- 相电感 Ls = 0.65 mH（d轴q轴相等）
- 反电动势常数 Ke = 0.15 V/(rad/s)

**硬件平台信息：**
- MCU：STM32G474（170MHz主频）
- PWM频率：可配置（支持20kHz/40kHz/80kHz）
- ADC采样：PWM中心对齐，单电阻采样
- 电流传感器：霍尔效应传感器，带宽100kHz

**设计要求：**
1. 选择合适的PWM频率并说明理由（2分）
2. 设计电流环带宽并计算PI参数（3分）
3. 验证设计的合理性（带宽约束、上升时间、稳定性）（3分）

**参考答案与评分标准：**

**(1) PWM频率选择（2分）**

**推荐选择：20kHz 或 40kHz（给1分）**

**理由（再给1分，至少说出2点）：**
- 20kHz：优点是开关损耗适中、EMI可控；缺点是电流纹波相对较大
- 40kHz：优点是电流纹波小、控制精度高；缺点是开关损耗大、对驱动电路要求高
- 80kHz：对于750W电机来说过高，开关损耗太大，性价比低

综合考虑750W功率等级和通用性，**选择20kHz**是合理的（若选40kHz也可接受，需给出理由）。

**(2) 带宽设计与参数计算（3分）**

**步骤1：确定带宽（1分）**

选择安全裕度因子为12（考虑到单电阻采样精度略差）：
$$\alpha = \frac{2\pi \times 20000}{12} = 10472 \text{ rad/s} \approx 1667 \text{ Hz}$$

或者选择更保守的因子15：
$$\alpha = \frac{2\pi \times 20000}{15} = 8378 \text{ rad/s} \approx 1333 \text{ Hz}$$

（写出计算过程即可得1分）

**步骤2：计算PI参数（2分）**

以α=10472为例：
$$K_p = \alpha \times L = 10472 \times 0.00065 = \mathbf{6.81 \text{ V/A}}$$ （1分）

$$K_i = \alpha \times R = 10472 \times 0.22 = \mathbf{2303.8 \text{ V/(A·s)}}$$ （1分）

**(3) 设计验证（3分）**

**验证1：带宽约束检查（1分）**

- 最大允许带宽：2π×20000/10 = 12566 rad/s
- 设计带宽：10472 rad/s
- **结论：10472 < 12566，满足约束 ✅** （0.5分）

- 考虑延迟：假设总延迟T_delay = 1.5/PWM = 75μs
- 延迟限制：1/(3×75μs) = 4444 rad/s
- **⚠️ 问题：设计带宽超过延迟限制！**

（若学生指出此问题，额外加0.5分；若未指出，不扣分但需在评语中提醒）

**修正建议：** 若考虑延迟，应将带宽降至3000-4000 rad/s左右。

**验证2：响应速度评估（1分）**

$$\tau_{closed} = \frac{1}{\alpha} = \frac{1}{10472} = \mathbf{0.0955 \text{ ms}}$$

$$t_r \approx \frac{2.2}{\alpha} = \mathbf{0.21 \text{ ms}}$$

原始时间常数：τ = L/R = 0.00065/0.22 = **2.95 ms**

**加速比：2.95/0.0955 = 30.9倍** （0.5分）

**评价：** 响应速度很快，适合伺服应用。（0.5分）

**验证3：稳定性初步判断（1分）**

- 零极点对消后理论上一阶系统，无条件稳定 ✅
- 但需注意：
  - 实际存在延迟（可能导致相位裕度下降）
  - 参数变化（L随电流饱和变化）
  - 未建模动态（传感器延迟、滤波器）

**结论：** 设计在理论上是合理的，但实际应用中建议：

OperationMode detect_operation_mode() {

**平滑过渡算法（1.5分）：**

```c
// 关键：避免参数突变导致冲击
void smooth_pi_switch(float target_Kp, float target_Ki,
                       float transition_time_ms) {
    static float Kp_start, Ki_start;
    static int step_count, total_steps;

    if (mode_changed) {  // 检测到模式切换请求
        Kp_start = current_Kp;
        Ki_start = current_Ki;
        total_steps = transition_time_ms / CONTROL_PERIOD_MS;
        step_count = 0;
    }

    if (step_count < total_steps) {
        // 线性插值过渡
        float ratio = (float)step_count / total_steps;
        current_Kp = Kp_start + (target_Kp - Kp_start) * ratio;
        current_Ki = Ki_start + (target_Ki - Ki_start) * ratio;

        step_count++;
    } else {
        // 过渡完成
        current_Kp = target_Kp;
        current_Ki = target_Ki;
    }
}
```

**过渡注意事项（0.5分）：**

3. **抗饱和处理：** 切换过程中暂时放宽积分限幅，防止Windup

**参考答案与评分指引：**

**(1) 局限性分析（3分，每点1分）**

**局限性1：参数敏感性（必须答出的核心问题）**
- 零极点对消的前提是**精确知道L和R**
- 但实际上：
  - L随磁饱和变化（电流增大时L可能下降20-50%）
  - R随温度变化（铜线温度系数+0.4%/°C，温升50℃→R增20%）
  - 制造公差（±10%很常见）
- 一旦L或R变化，Ki/Kp ≠ R/L，对消条件被破坏
- 后果：系统不再是理想一阶，可能出现超调或振荡

**局限性2：未建模动态**
- 该方法假设被控对象是**纯一阶RL电路**
- 忽略了：
  - 采样延迟（ADC+计算+PWM更新，通常1-2个PWM周期）
  - 滤波器动态（电流传感器的低通滤波）
  - 功率器件的非线性（死区、管压降）
  - 逆变器母线电压波动
- 这些"寄生动态"在高带宽时会严重影响稳定性

**局限性3：单一性能指标**
- 只有一个可调参数α（带宽）
- 无法独立调节：
  - 上升时间 vs 超调量的权衡
  - 跟踪性能 vs 抗扰能力的权衡
  - 标称性能 vs 鲁棒性的权衡
- 本质上是"单自由度"设计方法

**其他合理答案（可替代上述任一点）：**
- 不适用于非线性显著的系统（如深度磁饱和区）
- 无法处理多变量耦合（需要额外的解耦设计）
- 离散化后性能可能与连续域设计偏差较大

```c
// 伪代码：在线参数辨识
every_control_cycle:
    // 注入微小激励信号（伪随机二进制序列PRBS）
    u_inject = PRBS_signal(amplitude=0.05*I_rated);

    // 使用RLS更新L和R的估计值
    update_RLS_estimator(u_total, i_measured, &L_est, &R_est);

    // 更新PI参数
    Kp = alpha * L_est;
    Ki = alpha * R_est;
```

---

**方向B：鲁棒/自适应控制（现代控制范畴，1-1.5分）**

**思路：**
- 使用H∞控制或μ综合，显式地建模参数不确定性
- 设计一个控制器，在所有可能的(L, R)变化范围内都稳定
- 或者使用滑模控制（SMC），对参数变化不敏感

**优点：** 理论上保证鲁棒性，无需在线辨识
**挑战：** 设计复杂，实现难度高，可能有抖振现象

**具体方案示例（H∞混合灵敏度）：**
```
设计目标：
- 最小化灵敏度函数S（提高抗扰能力）
- 最小化补灵敏度函数T（提高鲁棒性）
- 约束控制量u不超过限幅

权重函数选择：
- W_s(s)：性能权重（低频段要求高增益→好跟踪）
- W_t(s)：鲁棒性权重（高频段要求低增益→对不确定性不敏感）
- W_u(s)：控制权重（限制控制量幅度）
```

---

**方向C：增益调度（工程实用方法，1分）**

**思路：**
- 离线在不同工作点（不同电流、不同温度）设计多组PI参数
- 运行时根据当前工作状态查表选择或插值
- 类似于航空发动机的增益调度控制

**优点：** 实现简单，计算量小，工业界常用
**缺点：** 需要大量离线测试，插值可能不够平滑

**实现框架：**
```c
// 二维查找表：[电流索引][温度索引]
typedef struct {
    float current_points[10];  // 电流工作点
    float temp_points[5];      // 温度工作点
    float Kp_table[10][5];     // 对应的Kp值
    float Ki_table[10][5];     // 对应的Ki值
} PI_Schedule_Table;

float get_scheduled_Kp(float I_abs, float temperature) {
    // 双线性插值查找当前最优Kp
    return bilinear_interpolation(schedule_table.Kp_table,
                                   I_abs, temperature);
}
```

---

**方向D：数据驱动/智能优化（前沿方向，1-1.5分）**

**思路：**
- 使用强化学习（RL）智能体自动搜索最优PI参数
- 或使用贝叶斯优化、遗传算法等全局优化方法
- 让系统通过"试错"自我学习最佳参数

**优点：** 可能发现人类工程师想不到的优秀参数组合
**挑战：** 需要大量训练数据，安全性难以保证，解释性差

**创新方案示例（模型预测控制MPC+学习）：**
```
传统方法：
  固定PI参数 → 应对所有工况

改进方法：
  1. 建立电机的数据驱动模型（神经网络拟合）
  2. 使用MPC在每个控制周期求解有限时域最优控制序列
  3. 用强化学习训练MPC的代价函数权重（自动平衡多个目标）
  4. 系统根据工况自适应调整控制策略
```

---

**(3) 未来发展趋势（3分，开放性回答，鼓励发散思维）**

**趋势1：自动化与智能化参数整定（必答方向，1分）**

- **自整定PID**：在上电初始化阶段自动执行参数辨识实验，自动计算最优PI参数
- **例子：** 现有的VESC、ODrive已经具备基本的电机参数辨识功能
- **未来：** 结合AI，实现"一键式"自整定——用户只需说"我要快响应"或"我要稳运行"

**趋势2：数字孪生与虚拟调试（1分）**

- 在实际硬件制造之前，先在数字孪生模型上完成控制参数设计和验证
- 使用高保真度的电机+逆变器+负载模型
- 大幅缩短开发周期，降低硬件损坏风险
- **关键技术：** 实时仿真、FPGA-in-loop、硬件在环（HIL）

**趋势3：云端协同与边缘智能（0.5-1分）**

- **云端：** 收集海量运行数据，训练全局优化模型，定期推送参数更新
- **边缘侧（MCU本地）：** 轻量级推理引擎，实时微调参数
- **案例：** 某品牌伺服驱动器通过云平台收集全球数万台设备的运行数据，利用机器学习发现"在温度>60°C且负载>80%时应自动将带宽降低15%"这样的隐性规则

**趋势4：可解释AI与形式化验证（ bonus 方向，0.5分）**

- 不仅让AI给出参数，还要**解释为什么**这样设
- 使用形式化方法验证参数的安全性（证明在某些条件下不会失稳）
- 结合领域知识（物理约束）与数据驱动，提高可信度
