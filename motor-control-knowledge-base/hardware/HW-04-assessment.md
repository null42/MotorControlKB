## 一、参数计算题（★★☆☆☆）

### 题目1：PWM死区电压损失计算
**问题：** 已知PWM频率10kHz（中心对齐），死区时间500ns，母线电压48V，电机电感1mH，额定电流10A。请计算：
1. 死区导致的电压损失
2. 死区导致的稳态电流误差
3. 死区补偿量（占空比修正值）

**参考答案：**
1. ΔV_dead = (Td / Tpwm) × Vdc × sign(I) = (0.5μs / 50μs) × 48V = 0.48V（电流正向时）
2. ΔI = ΔV / (L × fsw) = 0.48 / (1mH × 10kHz) = 48mA（稳态，但电流过零点影响更大）
3. 补偿量 ΔD = Td / Tpwm = 0.5μs / 50μs = 0.01 = 1%。当I > 0时占空比+1%，I < 0时占空比-1%

### 题目2：PWM分辨率计算
**问题：** STM32G4时钟频率170MHz，PWM开关频率20kHz（中心对齐模式）。请计算：
1. ARR值和PWM分辨率
2. 最低非零占空比对应的电压（Vdc=48V）
3. 分辨率不足对电流的影响

**参考答案：**
1. ARR = fclk / (2 × fsw) = 170MHz / (2 × 20kHz) = 4250；分辨率 = 1/4250 ≈ 0.024%
2. 最低非零占空比 = 1/4250 = 0.024%；对应电压 = 48V × 0.024% = 11.3mV
3. 电压量化台阶 = 48V / 4250 = 11.3mV；电流纹波增量 = 11.3mV / (1mH × 20kHz) = 0.57mA → 对10A额定电流可忽略。但在低占空比时（如D=1/4250），电压从0跳到11.3mV，可能导致低速时电流台阶

### 题目3：中断优先级与控制周期确定性
**问题：** 已知控制频率20kHz（50μs周期），FOC算法执行25μs，通信中断执行30μs，ADC采样+DMA 3μs。请分析：
1. 设计中断优先级方案
2. 通信中断能否在控制周期空闲时完成？
3. 如何解决？

**参考答案：**
1. 优先级方案：过流比较器(0,0) > 控制定时器(1,0) > ADC/DMA完成(2,0) > 编码器(3,0) > 通信接收(4,0) > 看门狗(5,0) > 温度采样(6,0)
2. 通信中断30μs > 控制周期空闲时间 = 50μs - 25μs(算法) - 3μs(ADC) = 22μs → 不能在空闲时完成！通信中断会抢占控制中断
3. 解决方案：
   - (a) 通信使用DMA+环形缓冲区，中断仅处理协议解析，执行时间<5μs
   - (b) 通信中断优先级低于控制中断，确保控制周期不被打断
   - (c) 通信数据处理放在主循环中（非实时任务）

## 二、电路设计题（★★★☆☆）

### 题目4：ADC同步采样时序设计
**问题：** 设计一个FOC控制系统的ADC采样时序，PWM频率20kHz（中心对齐），双ADC同步采样Ia和Ib，DMA传输。请给出：
1. 完整时序图（从PWM谷底到FOC计算完成）
2. ADC触发方案
3. 总延迟计算

**参考答案：**
1. 时序：
   - T=0μs：PWM谷底（计数器=0）
   - T=-2μs：ADC提前触发（计数器接近0时）
   - T=0~0.5μs：ADC采样保持
   - T=0.5~1.3μs：ADC转换（12位，12.5个时钟周期@16MHz ≈ 0.78μs）
   - T=1.3~1.5μs：DMA传输完成
   - T=1.5~48μs：FOC算法计算
   - T=48~50μs：写入新CCR值
2. ADC触发方案：使用TIM4比较事件触发ADC，比较值设置在计数器接近0时（如ARR-34），确保采样点在谷底附近。双ADC同步模式：ADC1为主采Ia，ADC2为从采Ib，同时触发
3. 总延迟 ≈ 1.5μs（ADC采样+转换+DMA）。FOC算法25μs，总控制周期占用约26.5μs < 50μs ✓

### 题目5：过流保护电路设计
**问题：** 为一个48V/30A电机控制器设计过流保护电路，要求：短路保护响应<1μs，过流保护阈值60A。请给出：
1. 硬件保护方案
2. 比较器选型和阈值设计
3. MCU刹车输入配置

**参考答案：**
1. 硬件方案：采样电阻 → 电流检测放大器 → 硬件比较器 → MCU PWM刹车输入。响应路径：过流 → 比较器翻转 → TIM1_BKIN → PWM自动关断，延迟<1μs
2. 比较器选型：TLV3501（响应时间40ns，推挽输出）。阈值设计：采样电阻5mΩ，放大器增益G=15；60A时Vsense = 60×0.005 = 0.3V，Vout = 0.3×15 = 4.5V；比较器参考电压Vref = 4.5V（用电阻分压从5V获得）；迟滞50mV防止误触发
3. MCU刹车配置：
   ```c
   TIM1->BDTR |= TIM_BDTR_BKE;    // 使能刹车
   TIM1->BDTR |= TIM_BDTR_BKP;    // 刹车极性高有效
   TIM1->BDTR &= ~TIM_BDTR_AOE;   // 刹车后不自动恢复（需软件清除）
   // 刹车中断中记录故障代码
   ```

## 三、故障诊断题（★★★☆☆）

### 题目6：电流波形扇区尖峰——死区未补偿
**问题：** FOC控制电机，电流波形在每个扇区切换处出现尖峰，6次/电周期。低速低调制比时更明显。请分析：
1. 根本原因
2. 尖峰为什么在低速时更明显？
3. 死区补偿算法的关键难点

**参考答案：**
1. 根本原因：PWM死区导致输出电压畸变。死区期间，输出电压由电流方向决定（而非PWM指令），导致实际电压与指令电压偏差。偏差在电流过零点（扇区切换处）最显著
2. 低速时更明显的原因：(a) 低速时调制比低 → 死区占比大 → 电压畸变严重；(b) 低速时反电动势小 → 死区电压占比大；(c) 低速时电流小 → 电流过零区域占比大 → 死区影响范围更宽
3. 死区补偿难点：(a) 电流过零点方向不确定 → 无法确定补偿方向；(b) 需要设置电流阈值（死区），阈值太小则噪声导致误补偿，太大则过零区域补偿不足；(c) 温度变化导致开关时间变化 → 补偿量需动态调整

### 题目7：控制周期抖动诊断
**问题：** FOC控制电机，GPIO翻转测量控制周期95~105μs（标称100μs），电流环偶尔震荡，高速时更明显。请分析：
1. 可能的原因（至少3个）
2. 诊断步骤
3. 解决方案

**参考答案：**
1. 可能原因：(a) 高优先级中断抢占（如通信中断优先级配置错误）；(b) 临界区过长（__disable_irq使用不当）；(c) DMA总线竞争导致CPU等待；(d) Flash缓存未命中
2. 诊断步骤：
   - (a) 检查NVIC优先级配置：确认控制中断优先级高于通信
   - (b) 测量各中断执行时间：GPIO翻转法测量每个中断的入口到出口时间
   - (c) 搜索代码中__disable_irq调用：确认临界区是否过长
   - (d) 使用逻辑分析仪分析中断时序
3. 解决方案：
   - (a) 调整中断优先级：控制中断为最高软件优先级
   - (b) 通信使用DMA，减少中断执行时间
   - (c) 缩短临界区，使用原子操作替代全局中断禁用
   - (d) 关键代码（FOC算法）放RAM执行，避免Flash等待

## 四、软硬件关联题（★★★★☆）

### 题目8：死区补偿算法设计
**问题：** 一个48V电机控制器，死区时间500ns，PWM频率20kHz。请设计完整的死区补偿算法：
1. 补偿量计算
2. 电流过零点处理策略
3. 温度补偿考虑
4. 补偿效果的验证方法

**参考答案：**
1. 补偿量：ΔD = Td / Tpwm = 0.5μs / 25μs = 0.02（中心对齐Tpwm=25μs半周期）；ΔV = ΔD × Vdc = 0.02 × 48 = 0.96V
2. 电流过零点处理：
   ```c
   if (I_phase > I_threshold) {
       D_comp = +ΔD;       // 正向电流
   } else if (I_phase < -I_threshold) {
       D_comp = -ΔD;       // 反向电流
   } else {
       // 过零区域：线性插值
       D_comp = ΔD * (I_phase / I_threshold);
   }
   ```
   I_threshold选择：通常取额定电流的2-5%，如10A额定取0.2-0.5A
3. 温度补偿：开关时间随温度变化 → 实际死区 = 设定死区 + 开关时间偏差。可在不同温度下标定补偿量，运行时查表
4. 验证方法：
   - (a) 示波器对比补偿前后电流波形，观察扇区尖峰是否减小
   - (b) FFT分析电流频谱，6次谐波应显著降低
   - (c) 低速平稳性测试：1rpm运行应无明显抖动

### 题目9：DMA双缓冲与CPU负载优化
**问题：** 一个FOC控制系统，控制频率20kHz，ADC采样3通道（Ia, Ib, Vbus），每通道12位。请分析：
1. 无DMA时CPU的采样开销
2. DMA双缓冲方案设计
3. DMA对CPU负载的改善量化

**参考答案：**
1. 无DMA：每次ADC转换完成后CPU需读取数据 → 3次读取 × (2μs等待+0.5μs读取) = 7.5μs/周期 → 占50μs周期的15%
2. DMA双缓冲方案：
   - 缓冲区A[3]和缓冲区B[3]，循环模式
   - DMA半传输中断 → 处理缓冲区A前半部分
   - DMA传输完成中断 → 处理缓冲区B后半部分
   - ADC配置为扫描模式（3通道），连续转换，DMA循环传输
3. CPU负载改善：
   - 无DMA：ADC处理7.5μs + FOC算法25μs = 32.5μs → CPU负载65%
   - 有DMA：ADC处理0μs（DMA自动完成）+ FOC算法25μs = 25μs → CPU负载50%
   - 节省7.5μs → 可用于更复杂算法或提高控制频率

### 题目10：MCU外设协同时序优化
**问题：** 一个FOC控制系统的完整时序如下：PWM更新→ADC触发→DMA传输→FOC计算→PWM写入。请分析：
1. 从PWM更新到下一次PWM更新的完整延迟链
2. 如何最小化ADC采样到PWM更新的延迟？
3. 中心对齐模式相比边沿对齐模式的优势（定量分析）

**参考答案：**
1. 延迟链：PWM更新(0) → 计数器上升+下降(0.5×Tpwm) → ADC触发(+触发延迟) → 采样保持(+0.5μs) → ADC转换(+0.8μs) → DMA传输(+0.2μs) → FOC计算(+25μs) → PWM写入(+0) → 下一周期生效
2. 最小化延迟的方法：
   - (a) ADC提前触发：在计数器接近0时触发，而非等计数器到0
   - (b) 缩短ADC采样时间（在保证精度前提下）
   - (c) DMA传输替代CPU轮询
   - (d) FOC算法优化（查表替代三角函数，定点替代浮点）
   - (e) PWM更新在FOC计算完成后立即写入CCR（影子寄存器在下一周期生效）
3. 中心对齐优势：
   - 边沿对齐：ADC只能在PWM周期开始时触发 → 延迟 = 0.5×Tpwm = 25μs（20kHz时）
   - 中心对齐：ADC可在谷底触发 → 延迟 ≈ 0（提前触发时）
   - 延迟减小25μs → 电流环带宽可提高约2倍
   - 中心对齐PWM纹波更小（对称波形）
