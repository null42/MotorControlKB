# MEC-03 多圈编码器

> 路径：⚙️ 传动与编码器 > MEC-03

**难度：** ★★★★☆

## 概述

多圈绝对值编码器（Multi-Turn Absolute Encoder）是伺服系统中实现"上电即知位置"的核心传感器。与单圈绝对值编码器仅能在一个旋转周期（0°~360°）内提供唯一位置值不同，多圈编码器还能记录旋转圈数，即使在断电后也能保持位置信息，彻底消除了伺服系统上电回零（homing）的需求。

对于电机控制工程师而言，多圈绝对值编码器的选型直接决定了伺服系统的上电行为、位置控制精度和通信实时性。理解其实现方案（Wiegand、齿轮计数、电池备份）和接口协议（SSI、BiSS、EnDat 2.2）的差异，是正确选型和驱动开发的基础。

## 正文

### 1. 单圈 vs 多圈绝对值编码器

#### 1.1 基本概念

| 特性 | 单圈绝对值 | 多圈绝对值 |
|------|-----------|-----------|
| 位置范围 | 0°~360°（一圈内唯一） | 0°~360° × N圈（全程唯一） |
| 断电后位置保持 | 保持单圈位置 | 保持单圈位置 + 圈数 |
| 上电后是否需要回零 | 需要回零确定圈数 | 无需回零，直接可用 |
| 典型分辨率 | 13~20 bit/圈 | 13~20 bit/圈 + 12~14 bit圈数 |
| 成本 | 较低 | 较高 |
| 适用场景 | 有限角度运动 | 多圈旋转运动（丝杠、卷绕等） |

#### 1.2 为什么需要多圈

在许多伺服应用中，负载运动范围超过一圈：

- **丝杠传动：** 电机旋转多圈对应直线位移全程
- **卷绕机构：** 电机连续旋转多圈
- **机器人关节：** 关节角度范围可能超过360°
- **电梯/提升机构：** 行程对应多圈旋转

若使用单圈编码器，断电后无法确定电机处于哪一圈，必须执行回零操作（homing），这在很多工业场景中不可接受（如断电后工件仍在夹具中，回零可能损坏工件）。多圈编码器解决了这一问题。

### 2. 多圈实现方案

#### 2.1 Wiegand传感器方案

**原理：** 利用韦根效应（Wiegand Effect）实现无源多圈计数。

Wiegand传感器由一段经过特殊处理的铁磁合金丝（Wiegand丝）和感应线圈组成。当外部磁场方向反转时，Wiegand丝内部的磁畴发生突变翻转（Barkhausen跳变），在线圈中产生一个幅度和宽度恒定的脉冲电压（约1~3V，脉宽10~20μs），与旋转速度无关。

**实现方式：**
1. 在编码器主轴上安装多极磁环（如4极、8极）
2. 主轴每转一圈，磁极反转次数 = 极对数
3. Wiegand传感器检测每次磁极反转，产生计数脉冲
4. 计数脉冲驱动低功耗计数器（CMOS电路），累计圈数
5. 断电时，Wiegand脉冲的能量足以维持计数器工作

**关键特性：**

| 参数 | 典型值 |
|------|--------|
| 工作电压 | 无需外部供电（自供电计数） |
| 计数速度 | 最高约 12000 rpm（受磁极反转频率限制） |
| 圈数位数 | 12~14 bit（4096~16384圈） |
| 电池需求 | 无（真正的无电池多圈） |
| 工作温度 | -40°C ~ +100°C |
| 寿命 | 无磨损，理论无限 |

**优势：**
- 无需电池，免维护
- 断电后仍可记录最多约 ±2000 圈（取决于具体实现）
- 不受转速影响（低速和高速均产生等幅脉冲）

**局限：**
- Wiegand丝的磁畴翻转有方向性，需正确安装极性
- 计数器需极低功耗设计（nA级静态电流）
- 振动环境下可能产生误计数（需软件滤波）

**代表产品：** Renishaw RESOLUTE™、SICK DBS60E、Baumer EIL580。

#### 2.2 齿轮计数机构方案

**原理：** 采用纯机械齿轮组实现多圈计数，类似钟表机构。

**实现方式：**
1. 主轴驱动一组精密减速齿轮
2. 齿轮组的末级连接一个磁编码器或光电编码器
3. 齿轮减速比决定了圈数范围
4. 各级齿轮位置组合编码，得到唯一的多圈位置值

**示例：** 若齿轮组为3级，每级速比16:1，则圈数范围 = $16^3 = 4096$ 圈（12 bit）。

**关键特性：**

| 参数 | 典型值 |
|------|--------|
| 圈数位数 | 12~14 bit |
| 电池需求 | 无（纯机械） |
| 机械寿命 | 约 $10^7$ 圈（轴承磨损限制） |
| 响应速度 | 最高约 12000 rpm |
| 体积 | 较大（齿轮组占用空间） |

**优势：**
- 完全无电池、无电子计数
- 可靠性极高（不受电磁干扰）
- 断电后位置永久保持

**局限：**
- 齿轮磨损影响精度
- 体积较大
- 齿轮背隙可能导致计数误差
- 高速旋转时齿轮惯性大

**代表产品：** Kübler Sendix 5868、FRABA Posital OCD-PI。

#### 2.3 电池备份SRAM方案

**原理：** 使用增量式或单圈绝对值编码器 + 低功耗计数器 + 电池供电SRAM实现多圈记忆。

**实现方式：**
1. 编码器内部有单圈绝对值位置检测
2. 单圈过零点时产生方向信号，驱动硬件计数器
3. 计数值存储在SRAM中，由锂电池维持
4. 上电时读取SRAM中的圈数 + 当前单圈位置 = 完整位置

**关键特性：**

| 参数 | 典型值 |
|------|--------|
| 圈数位数 | 12~14 bit |
| 电池寿命 | 5~10年（取决于电池容量和静态功耗） |
| 电池类型 | CR2032或专用锂亚电池 |
| 静态功耗 | 0.1~1 μA（低功耗SRAM） |

**优势：**
- 结构简单，成本低
- 技术成熟，应用广泛
- 体积小

**局限：**
- **电池寿命有限**，需定期更换
- **电池耗尽后丢失圈数**，必须回零
- 低温环境下电池性能下降
- 电池更换期间位置信息丢失
- 某些行业（食品、医药）禁止使用电池

**代表产品：** Heidenhain ROC/ROQ系列（部分型号）、Tamagawa TS5700N系列。

#### 2.4 三种方案对比

| 对比项 | Wiegand | 齿轮计数 | 电池SRAM |
|--------|---------|---------|---------|
| 是否需要电池 | 否 | 否 | 是 |
| 维护需求 | 无 | 低（机械磨损） | 定期换电池 |
| 可靠性 | 高 | 高 | 中（电池风险） |
| 体积 | 小 | 较大 | 小 |
| 成本 | 中 | 高 | 低 |
| 圈数范围 | 12~14 bit | 12~14 bit | 12~14 bit |
| 高速适应性 | 好 | 一般 | 好 |
| 断电后计数能力 | ±2000圈 | 永久 | 电池耗尽前 |

**选型建议：**
- 免维护、高可靠性 → Wiegand方案
- 极端环境（强电磁干扰）→ 齿轮计数方案
- 成本敏感、可接受定期维护 → 电池SRAM方案

### 3. 接口协议详解

#### 3.1 SSI（同步串行接口）

SSI（Synchronous Serial Interface）是最基本的绝对值编码器通信协议，由德国Stegmann公司（现Heidenhain旗下）开发。

**物理层：**
- 4线制：CLK+、CLK-、DATA+、DATA-（RS-422差分传输）
- 2线制：CLK、DATA（单端，抗干扰差，不推荐工业使用）

**通信时序：**

```
主机发送时钟          ┌─┐ ┌─┐ ┌─┐ ┌─┐       ┌─┐ ┌─┐
CLK ────────────────┘ └─┘ └─┘ └─┘ └─┘ ... └─┘ └─┘ └──
                     │ │ │ │ │ │ │ │         │ │ │ │
编码器响应     MSB ──┤Dn├─┤Dn├─┤Dn├─┤Dn├─...─┤D1├─┤D0├──
DATA ────────────────┘ └─┘ └─┘ └─┘ └─┘       └─┘ └─┘
                     ◄── 单稳态触发延迟 ──►
```

**帧结构：**
1. 主机拉低CLK至少 $t_{idle}$（空闲时间，通常 > 20μs）
2. 第一个CLK上升沿：编码器锁存当前位置值
3. 后续每个CLK下降沿：编码器输出1 bit数据（MSB优先）
4. 数据位数 = 单圈位数 + 多圈位数 + 状态位（通常24~32 bit）
5. 传输完成后，DATA线保持高电平

**关键参数：**

| 参数 | 典型值 |
|------|--------|
| 时钟频率 | 100 kHz ~ 2 MHz |
| 空闲时间 $t_{idle}$ | > 20 μs |
| 单稳态延迟 $t_{mon}$ | 10~25 μs |
| 数据位数 | 24/25/32 bit |
| 传输延迟 | 数十μs |

**SSI的局限：**
- 单向通信（编码器→主机），无法写入参数
- 无CRC校验（部分扩展版SSI支持）
- 传输速率有限
- 不支持多编码器总线

#### 3.2 BiSS（双向串行接口）

BiSS（Bidirectional Serial Interface）是德国iC-Haus公司开发的开放协议，在SSI基础上增加了双向通信和CRC校验。

**物理层：**
- 4线RS-422：MA+、MA-（主机时钟）、SL+、SL-（从机数据）
- 支持总线拓扑（多编码器菊花链连接）

**通信模式：**

**1) BiSS-C（周期模式，最常用）：**

```
MA  ──┐ ┌─┐ ┌─┐ ┌─┐ ┌─┐       ┌─┐ ┌─┐ ┌─┐ ┌─┐
      └─┘ └─┘ └─┘ └─┘ └─┘ ... └─┘ └─┘ └─┘ └─┘
SL  ─────┐ ┌─┐ ┌─┐ ┌─┐ ┌─┐       ┌─┐ ┌─┐ ┌─┐ ┌─┐
      ACK└─┘ └─┘ └─┘ └─┘ └─┘ ... └─┘ └─┘ └─┘ └─┘
      │◄─ ACK ─►│◄──── 数据 (MSB first) ────►│◄CRC►│
```

帧结构：
- **ACK：** 编码器响应信号（1 bit）
- **数据：** 单圈 + 多圈位置值（可配置长度）
- **CRC：** 6 bit多项式校验（$x^6 + x^1 + 1$）
- **Error/Warning：** 2 bit状态位

**2) BiSS-C寄存器模式：**
- 支持读写编码器内部寄存器
- 可配置分辨率、方向、预分频等参数
- 支持电子铭牌（Electronic Nameplate，含编码器型号、序列号）

**关键参数：**

| 参数 | 典型值 |
|------|--------|
| 时钟频率 | 最高 10 MHz |
| 传输延迟 | < 10 μs（典型3~5 μs） |
| 数据更新率 | 最高 20 kHz+ |
| CRC校验 | 6 bit |
| 总线节点数 | 理论无限制（受延迟约束） |
| 开放性 | 完全开放，无授权费 |

**BiSS优势：**
- 开放协议，无授权费用
- 高传输速率（10 MHz时钟）
- CRC校验保证数据完整性
- 双向通信，支持寄存器读写
- 支持总线拓扑
- 兼容SSI物理层（可平滑升级）

**代表芯片：** iC-Haus iC-MU（编码器端）、iC-Haus iC-PZ（主机端）。

#### 3.3 EnDat 2.2（海德汉）

EnDat（Encoder Data Interface）2.2 是德国Heidenhain公司的专有协议，支持增量+绝对值混合传输，是目前分辨率最高的编码器接口之一。

**物理层：**
- 4线RS-422/RS-485：CLK+、CLK-、DATA+、DATA-
- 8线制（含增量信号）：CLK±、DATA±、A±、B±（增量信号独立传输）

**通信模式：**

**1) 周期模式（Position Mode）：**

```
CLK ──┐ ┌─┐ ┌─┐ ┌─┐       ┌─┐ ┌─┐ ┌─┐ ┌─┐ ┌─┐ ┌─┐
      └─┘ └─┘ └─┘ └─┘ ... └─┘ └─┘ └─┘ └─┘ └─┘ └─┘
DATA ──────┐ ┌─┐ ┌─┐       ┌─┐ ┌─┐ ┌─┐ ┌─┐ ┌─┐ ┌─┐
      Mode └─┘ └─┘ └─┘ ... └─┘ └─┘ └─┘ └─┘ └─┘ └─┘
      │◄─ Mode ─►│◄── 绝对值数据 ──►│◄CRC►│◄Error►│
```

帧结构：
- **Mode位：** 2~3 bit，指定传输模式（位置/参数/复位）
- **绝对值数据：** 单圈 + 多圈位置值
- **CRC：** 5 bit多项式校验
- **Error位：** 1~2 bit状态位

**2) 参数模式（Parameter Mode）：**
- 支持读写编码器内部存储器
- 可读取电子铭牌信息
- 可配置分辨率、方向、警告阈值等

**3) 增量信号叠加：**
- EnDat 2.2可同时传输增量信号（1 Vpp正弦/余弦信号）
- 增量信号提供高分辨率细分（插值后可达29 bit+）
- 绝对值信号提供粗定位，增量信号提供精定位

**关键参数：**

| 参数 | 典型值 |
|------|--------|
| 时钟频率 | 100 kHz ~ 16 MHz |
| 传输延迟 | < 5 μs（16 MHz时钟时） |
| 最大分辨率 | 29 bit（含增量细分） |
| CRC校验 | 5 bit |
| 供电电压 | 3.6V ~ 5.25V / 7V ~ 12V |
| 电缆长度 | 最长 150m（低时钟频率时） |

**EnDat 2.2优势：**
- 超高分辨率（29 bit+）
- 增量+绝对值混合传输
- 完善的参数配置能力
- Heidenhain生态完整
- 高可靠性（工业验证数十年）

**EnDat 2.2局限：**
- 专有协议，需授权
- 编码器价格高
- 与第三方驱动器兼容性需确认

**代表产品：** Heidenhain ROC 425、ROQ 437、ERN 1387。

#### 3.4 三种协议对比

| 对比项 | SSI | BiSS-C | EnDat 2.2 |
|--------|-----|--------|-----------|
| 通信方向 | 单向 | 双向 | 双向 |
| 时钟频率 | ≤ 2 MHz | ≤ 10 MHz | ≤ 16 MHz |
| CRC校验 | 无/可选 | 6 bit | 5 bit |
| 寄存器读写 | 不支持 | 支持 | 支持 |
| 增量信号 | 不支持 | 不支持 | 支持 |
| 总线拓扑 | 不支持 | 支持 | 不支持 |
| 开放性 | 开放 | 开放 | 专有 |
| 最大分辨率 | ~24 bit | ~28 bit | ~29 bit |
| 传输延迟 | ~50 μs | ~5 μs | ~5 μs |
| 成本 | 低 | 中 | 高 |

**其他协议：**
- **HIPERFACE DSL®**（SICK/Stegmann）：单电缆方案（供电+通信共用一对线），节省布线
- **Tamagawa SmartAbs**：安川/松下等日系驱动器常用
- **Nikon S-Format**：尼康专有协议
- **SPI**：简单直接，适合嵌入式系统内部通信

### 4. 分辨率与精度

#### 4.1 分辨率（Resolution）

分辨率是编码器能区分的最小角度单位，由位数决定：

$$\text{分辨率} = \frac{360°}{2^N}$$

| 单圈位数 | 角度分辨率 | 对应弧度 |
|---------|-----------|---------|
| 13 bit | 0.0439° | 0.767 mrad |
| 16 bit | 0.00549° | 95.9 μrad |
| 17 bit | 0.00275° | 47.9 μrad |
| 20 bit | 0.000343° | 5.99 μrad |
| 23 bit | 0.0000429° | 0.749 μrad |

**多圈总分辨率：**

$$\text{总位数} = \text{单圈位数} + \text{多圈位数}$$

例如：单圈20 bit + 多圈12 bit = 32 bit总分辨率，可表示 $2^{20} \times 2^{12} = 4,294,967,296$ 个唯一位置。

#### 4.2 精度（Accuracy）

精度是编码器输出值与真实角度的偏差，通常用角秒（arcsec）表示：

$$1 \text{ arcsec} = \frac{1^\circ}{3600} \approx 4.848 \text{ μrad}$$

**分辨率 ≠ 精度：** 分辨率是可分辨的最小步距，精度是测量值的正确程度。一个20 bit分辨率的编码器精度可能只有 ±20 arcsec。

| 编码器等级 | 单圈精度 | 典型应用 |
|-----------|---------|---------|
| 经济型 | ±120 arcsec | 通用自动化 |
| 标准型 | ±60 arcsec | 数控机床 |
| 精密型 | ±20 arcsec | 精密转台 |
| 超精密型 | ±5 arcsec | 半导体设备、计量 |

**精度影响因素：**
- 码盘制造精度（光栅刻划误差、磁极分布误差）
- 轴承偏心和跳动
- 安装误差（偏心、倾斜）
- 温度漂移
- 信号处理误差（插值非线性）

### 5. 选型要点

#### 5.1 分辨率选择

根据系统定位精度需求反推：

$$N_{min} = \lceil \log_2 \frac{360° \times 3600}{\theta_{accuracy} \times S} \rceil$$

其中 $\theta_{accuracy}$ 为要求的定位精度（arcsec），$S$ 为安全裕度（通常取2~5）。

**示例：** 要求定位精度 ±10 arcsec，安全裕度3：

$$N_{min} = \lceil \log_2 \frac{1296000}{10 \times 3} \rceil = \lceil \log_2(43200) \rceil = \lceil 15.4 \rceil = 16 \text{ bit}$$

考虑减速器传动比 $i$，电机端编码器分辨率需求可降低：

$$N_{motor} = N_{min} - \log_2(i)$$

#### 5.2 精度选择

编码器精度需满足：

$$\varepsilon_{encoder} \leq \frac{\varepsilon_{system}}{K}$$

其中 $\varepsilon_{system}$ 为系统总精度要求，$K$ 为精度分配系数（通常取2~3，考虑其他误差源贡献）。

#### 5.3 响应速度

编码器数据更新率需满足伺服控制环路的需求：

$$f_{encoder} \geq k \times f_{loop}$$

其中 $f_{loop}$ 为位置环/速度环频率，$k$ 为过采样系数（通常取3~5）。

**典型要求：**
- 位置环 1 kHz → 编码器更新率 ≥ 3~5 kHz
- 速度环 4 kHz → 编码器更新率 ≥ 12~20 kHz
- 电流环 16 kHz → 通常使用增量信号或霍尔信号

#### 5.4 环境等级

| 防护等级 | 适用环境 |
|---------|---------|
| IP40 | 干净室内 |
| IP64 | 一般工业 |
| IP65 | 有粉尘、溅水 |
| IP67 | 有油污、潮湿 |
| IP69K | 高压冲洗环境 |

**其他环境参数：**
- 工作温度范围（-40°C ~ +100°C / -20°C ~ +85°C）
- 抗振动（10~2000 Hz，≤ 200 m/s²）
- 抗冲击（≤ 1000 m/s²）
- EMC等级（IEC 61000）

#### 5.5 接口兼容性

选型时需确认编码器接口与伺服驱动器的兼容性：

| 驱动器品牌 | 支持的编码器协议 |
|-----------|---------------|
| 倍福 | BiSS、EnDat、SSI |
| 西门子 | EnDat 2.2、SSI |
| 安川 | Tamagawa、绝对值编码器专有协议 |
| 松下 | Tamagawa、NIKON |
| 汇川 | BiSS、EnDat、SSI、Tamagawa |
| 禾川 | BiSS、SSI |

### 6. 在伺服系统中的作用

#### 6.1 上电即知位置——消除回零

多圈绝对值编码器最核心的价值是**消除上电回零**：

- **传统增量编码器：** 上电后位置未知，必须执行回零操作（寻找零点开关或限位），耗时且可能干涉工件
- **单圈绝对值编码器：** 上电后知道0°~360°内的位置，但不知道圈数，仍需回零确定绝对位置
- **多圈绝对值编码器：** 上电后直接获得完整位置（单圈位置 + 圈数），无需任何回零操作

**回零消除的价值：**
- 减少开机时间（从数十秒降至零）
- 避免回零过程中碰撞工件
- 断电恢复后立即恢复生产
- 简化PLC程序逻辑

#### 6.2 寻相（Phase Finding）

在永磁同步电机（PMSM/BLDC）的FOC控制中，需要知道转子磁极的绝对位置（电角度）才能正确进行Park变换和SVPWM调制。

**无多圈编码器时：** 需执行寻相操作（如高频注入法、定子对齐法），耗时且可能引起电机抖动。

**有多圈编码器时：** 编码器安装时标定机械零位与电角度零位的偏移量，上电后直接计算电角度：

$$\theta_e = (\theta_{mech} - \theta_{offset}) \times p$$

其中 $\theta_{mech}$ 为编码器机械角度，$\theta_{offset}$ 为标定偏移量，$p$ 为极对数。

**注意事项：**
- 编码器与电机轴的安装偏心会导致电角度误差
- 多极对电机（$p > 1$）对角度误差更敏感
- 建议在电机装配后进行一次性标定，将偏移量写入驱动器参数

#### 6.3 位置环反馈

多圈编码器作为位置环反馈元件，直接影响定位精度：

$$\varepsilon_{position} = \varepsilon_{encoder} + \varepsilon_{mechanical} + \varepsilon_{control}$$

其中：
- $\varepsilon_{encoder}$：编码器自身精度
- $\varepsilon_{mechanical}$：传动系统误差（背隙、弹性变形）
- $\varepsilon_{control}$：控制算法误差（跟随误差、稳态误差）

**双环控制方案：** 在电机端和负载端各安装一个编码器：
- 电机端编码器：用于电流环换相和速度环反馈
- 负载端编码器：用于位置环反馈

此方案可消除传动系统误差对定位精度的影响，但增加了成本和复杂度。

#### 6.4 速度测量

编码器位置信号经微分可得速度：

$$\omega = \frac{\Delta\theta}{\Delta t}$$

**M法（测频法）：** 在固定时间窗口内计数位置脉冲数，适合高速。

$$\omega = \frac{m}{T_s \times N_{ppr}} \times 60 \text{ (rpm)}$$

**T法（测周法）：** 测量相邻脉冲的时间间隔，适合低速。

$$\omega = \frac{f_{clock}}{n \times N_{ppr}} \times 60 \text{ (rpm)}$$

**M/T法：** 同时测量脉冲数和时间，全速域适用，是伺服驱动的标准方案。

高分辨率编码器（20 bit+）可在极低转速下仍提供足够的速度分辨率，实现平滑的低速运行。

## 小结

多圈绝对值编码器是现代伺服系统实现"上电即知位置"的关键传感器。三种多圈实现方案各有优劣：Wiegand方案免维护但成本中等，齿轮计数方案纯机械但体积大，电池SRAM方案成本低但有电池寿命风险。接口协议方面，SSI简单但功能有限，BiSS开放且高性能，EnDat 2.2专有但分辨率最高。选型时需从分辨率、精度、响应速度、环境等级和接口兼容性五个维度综合考量。在伺服系统中，多圈编码器不仅消除了回零需求，还简化了寻相过程，直接决定了位置控制精度和系统上电行为。
